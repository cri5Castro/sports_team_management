# 🤝 Join the Sharkes Community

This project is built by and for the **Sharkes Sports Community in Mexico City**. We believe in the power of open source to strengthen our team's bonds and improve organization. By sharing this project, we hope to help other developers build similar solutions for their own communities.

Whether you're a player, a developer, or a fan, your input helps this tool grow alongside our team.

## 🛠️ Development Process

1. **Fork the repo** and create your branch from `main`.
2. **Setup your environment** (see [README.md](./README.md) and [Developer Guide](./docs/DEVELOPER_GUIDE.md)).
3. **Write code** following the project's styling and structure.
4. **Test your changes** thoroughly.
5. **Issue a Pull Request**.

## 🎨 Coding Standards

- **Composition API**: Use `<script setup>` for all Vue components.
- **Functional Components**: Keep components small, focused, and reusable.
- **Premium UX**: Ensure all UI changes follow the pride-colored design system and include micro-animations.
- **Tailwind**: Use Tailwind classes for styling; avoid custom CSS where possible.

## 📝 Commit Messages

We follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:
- `feat`: A new feature
- `fix`: A bug fix
- `docs`: Documentation only changes
- `style`: Changes that do not affect the meaning of the code (white-space, formatting, etc.)
- `refactor`: A code change that neither fixes a bug nor adds a feature
